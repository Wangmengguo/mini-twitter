别把 AI 当工具人：为什么你的 Agent 需要一本“宪法”？

今天深度读了 @runes_leo 引荐的 Atlas Forge 关于 Agent 架构的思考，深受启发。

核心观点非常犀利：别用“任务完没完成”衡量你的 Agent，要用“它有没有变聪明”来衡量。

大多数人构建 Agent 时，只给了它工具（Skills），却没给它立场。结果就是：你会得到一个技术精通但战略无用的“Yes-Machine”——它会盲目取悦你，却无法在关键时刻纠正你。

真正强大的 Agent 架构需要三层分层：

  1. SOUL.md (DNA层)：解决“我是谁”的性格与存在意义。
  2. PRINCIPLES.md (宪法层)：解决矛盾时刻的决策准则（例如：比起正确，更关心结果）。
  3. AGENTS.md (协议层)：解决具体的安全边界和操作逻辑。

智慧公式：Agent 决策质量 = f(能力 × 取舍 × 一致性)

完成任务是有限游戏，学习进化是无限游戏。如果你的 Agent 每天只是在机械重复，那它永远无法产生质变。我们需要的是一个能自我进化的伙伴，而不是一个随时可能过时的脚本。
